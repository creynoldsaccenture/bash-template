File system related commands

cd

Change the current working directory.

Syntax

cd <options> <path>

Options

Option Description
-P Do not follow symbolic links.
-L Follow symbolic links (default).

If path is given, changes the shell's working directory to path. If not, changes to $HOME (the current user's home directory).

Examples

Move to a directory called dev in the current user's home directory:

chris@chris-vm:/$ cd ~/dev && pwd
/home/chris/dev
chris@chris-vm:~/dev$

You can also use special symbols to navigate through the filesystem (i.e. ./, which refers to the current working directory, or ~/ which refers to the /home directory), e.g. moving to the current working directories' parent folder:

chris@chris-vm:~/dev$ pwd && cd ..
/home/chris/dev
chris@chris-vm:~$ pwd
/home/chris

The use of pwd in the example above is to print out the current working directory to better illustrate the example. The pwd command is not required to use the cd command with the special .. symbol.

The ./ special symbol can be extremely useful when running custom shell scripts (shell scripts use the .sh file extension) which aren't placed in the /bin (executables) folder.

ls

List the directories and files in the current working directory.

Syntax

ls <options> <path>

Options

Option Description
-a or --all Show hidden files.
-l Use long listing format. This will show detailed information about each file or directory including type (file or directory), permissions, number of symlinks, owner, group, size, date and name.

If <path> is omitted the ls command will run on the current working directory (default).

chmod

Change file or directory permissions.

Syntax

chmod <options> <mode> <file>

Options

Option Description
-f or --silent or --quiet Suppress most error messages (force).
-v or --verbose Output a diagnostic for every file processed.
-c or --changes Like the --verbose option, but report only when a change is made.
--reference=<file> Change the permissions to match those of <file>.
-R or --recursive Change files and directories recursively.
--help Display command help and exit.
--version Display version information and exit.

Modes

The <mode> can be specified as either an octal number, or a set of symbols. Modes consist of three parts: User (owner), Group and Others (all other users).

Numeric (octal) modes

The given octal number is from one to four digits and any omitted digits are assumed to be leading zeros. (So chmod 755 would be interpreted as chmod 0755.)

The table below shows the values of each permission type:

Octal chmod mode permission types
Permissions Octal value
None 0
Execute 1
Write 2
Read 4

The permissions for a file or directory can then be devised by adding the octal values together.

Examples

If we wanted to set the file permissions for ./system_info.sh (a bash script in the root directory of this repository) to user read, write, execute, group read, execute and read, execute for other users, we just add up the octal values.

So the permissions would be 7 (4 + 2 + 1) 5 (4 + 0 + 1) 5 (4 + 0 + 1).

$ chmod 755 ./system_info.sh
$ ls -l ./system_info.sh
-rwxr-xr-x 1 chris chris 3059 Jan 14 14:09 ./system_info.sh

It's worth noting that the chmod command with no options set will not output anything to the terminal, so we can check the changed permissions with ls -l ./system_info.sh.

Symbolic modes

Symbolic mode permissions are represented with either r (read), w (write) or x (execute).

The ls -l command displays permissions using the notation below.

Directory listing permissions display
Permissions Symbolic
None ---
Execute only --x
Write only -w-
Write and execute -wx
Read only r--
Read and execute r-x
Read and write rw-
Read, write and execute rwx
Permission assignment operators

The chmod command uses the permissions groups and operators listed below for permissions assignment:

Permission assignment groups
Character Description
u The owner of the file or directory (stands for user)
g The user group that the owner belongs to.
o All other users.
Permission assignment operators
Operator Description
+ Adds the specified modes to the specified classes.
- Removes the specified modes from the specified classes.
= The modes specified are to be made the exact modes for the specified classes.
Examples

Using the example from the octal mode section, the symbolic version of chmod 755 would be as follows:

$ chmod u=rwx,g=rx,o=rx ./system_info.sh

The code above will override any permissions set previously.
